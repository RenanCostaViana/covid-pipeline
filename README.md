# 🦠 COVID Data Pipeline — Public Health Analytics

Pipeline de dados para análise de informações epidemiológicas da COVID‑19  
Projeto focado em ingestão, transformação e estruturação de dados públicos de saúde

---

## 🧠 Visão Geral

Este projeto apresenta um **pipeline de dados completo** aplicado ao contexto da
**pandemia de COVID‑19**, com foco na coleta, organização, tratamento e análise de
dados epidemiológicos provenientes de fontes públicas.

O objetivo principal é demonstrar a **estruturação de um fluxo analítico confiável**,
capaz de transformar dados brutos em bases organizadas e prontas para análise,
seguindo boas práticas de engenharia e análise de dados.

> ⚠️ Este projeto tem caráter **analítico e educacional** e não deve ser utilizado
para diagnóstico clínico ou tomada de decisão médica.

---

## 🎯 Objetivos do Projeto

- Realizar ingestão de dados públicos relacionados à COVID‑19
- Estruturar um pipeline de dados reprodutível
- Limpar, padronizar e tratar informações epidemiológicas
- Consolidar indicadores relevantes para análise
- Demonstrar boas práticas de organização de dados públicos
- Servir como base para análises exploratórias e dashboards

---

## 🔁 Etapas do Pipeline

1. **Ingestão de Dados**
   - Coleta de dados epidemiológicos a partir de fontes públicas
   - Estruturação inicial das bases
   - Padronização de formatos e schemas

2. **Tratamento e Transformação**
   - Limpeza de inconsistências
   - Tratamento de valores ausentes
   - Padronização de métricas e datas
   - Criação de colunas derivadas

3. **Organização Analítica**
   - Estruturação dos dados para consumo analítico
   - Preparação de tabelas consolidadas
   - Base pronta para EDA ou visualização

4. **Análise de Indicadores**
   - Casos confirmados
   - Óbitos
   - Evolução temporal
   - Comparações e tendências

---

## 🧠 Principais Aprendizados

- Dados públicos exigem forte atenção à qualidade e padronização
- Estrutura de pipeline é tão importante quanto a análise final
- Contextos críticos exigem interpretação responsável
- Separar ingestão, tratamento e análise facilita manutenção e reuso
- Pipelines bem estruturados podem ser facilmente adaptados a outros domínios

---

## ⚠️ Limitações

- O projeto não contempla dados clínicos individuais
- Os dados dependem da qualidade e atualização das fontes públicas
- Não há modelagem preditiva nem uso para decisões médicas
- O pipeline tem caráter analítico, não operacional em produção
